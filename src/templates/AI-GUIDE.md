# AI Guide — Knowledge Vault Structure

This file defines the vault structure and navigation rules for AI agents.

---

## Vault Structure

```
vault/
├── AI-GUIDE.md              ← This file. Structure + navigation.
├── MY-PROFILE.md            ← User profile (generated by kore-chamber init)
├── 00-Inbox/                ← Unclassified
├── 10-Concepts/             ← "What is X?"
├── 20-Troubleshooting/      ← "Error → Cause → Fix"
├── 30-Decisions/            ← "Why B instead of A?"
├── 40-Patterns/             ← Reusable implementation methods
├── 50-MOC/                  ← Map of Contents (domain indexes)
└── Templates/               ← Note templates (do not modify)
```

---

## Navigation Rules

1. **Start from MOC.** `50-MOC/` indexes link all notes in a domain.
2. **Follow wiki-links.** Notes have `## 관련 노트` with related links at the bottom.
3. **Use summaries.** Each note has a `summary` field in frontmatter — read this before the full body.
4. **Ignore Templates/.**

## Tag System

Tags represent **technology domains** only. Note type is determined by folder.

---

## File Naming Rules

- Default language of the user, hyphen-separated
- Industry-standard abbreviations stay in English: DB, API, MSA, JWT, etc.
- No type words in filename (folder handles that)
- Length: 3–5 words

---

## Note Format

```yaml
---
created: YYYY-MM-DD
tags: [domain]
type: concept/troubleshooting/decision/pattern/inbox
summary: "One sentence description"
---
# Title

[Free-form body]

## 관련 노트
- [[linked-note]]
```
